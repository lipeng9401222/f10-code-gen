## 一、什么是 SPA？

SPA，全称为 **Single-Page Application（单页应用程序）**，是一种特殊的 Web 应用开发架构。与您熟悉的“每个页面都是一个独立 HTML 文件”的传统 Web 应用不同，SPA **从服务器加载一个初始的 HTML 页面后，后续的页面跳转几乎不再重新请求完整的新 HTML 页面**。

取而代之的是，SPA 通过 JavaScript 动态地获取到浏览器地址的变化，然后根据当前的 URL 路径，使用不同的内容来更新当前页面的内容、样式和数据，使得用户在与应用交互时，感觉就像在使用一个本地桌面应用或移动 App 一样流畅，页面不会有传统意义上的“刷新”动作。

简单来说，**整个应用就像一个页面，但内部能够通过 JavaScript 动态切换不同的“视图”内容**。

至于客户端的 JavaScript 如何根据地址栏的变化，来动态加载和渲染不同的内容，这通常是通过[前端路由](./router.md)来实现的。

## 二、SPA 的主要特性

1.  **单页加载**：

    - 初始时，浏览器仅从服务器请求并加载一个主要的 HTML 文件（通常结构非常简单，包含应用容器）、必要的 CSS 和 JavaScript 文件。
    - 后续所有交互（如点击导航、提交表单等）不再触发新的完整 HTML 页面请求。

2.  **动态内容更新**：

    - 存在前端路由体系，会感知 URL 的变化，并根据路由规则动态加载和渲染不同的组件或模板。
    - 用户看到的“页面切换”，实际上是前端框架在同一个 HTML 页面内，根据路由规则渲染不同的组件或模板到指定容器中。

3.  **更优的用户体验 (UX)**：

    - **减少页面跳转和白屏时间**：避免了传统页面切换时的整页刷新和等待，用户操作反馈更快，体验更接近原生应用。
    - **流畅的过渡动画**：更容易实现页面内不同视图切换时的平滑过渡效果。

4.  **状态管理**：
    - 由于所有交互都在一个页面内，应用的状态可以更方便地在客户端进行管理和维护。

## 三、SPA 与传统多页面的 (MPA) 的区别

| 特性               | 传统多页面 (MPA)                                                       | SPA (单页应用)                                                 |
| :----------------- | :--------------------------------------------------------------------- | :------------------------------------------------------------- |
| **页面数量**       | 通常对应多个独立的 HTML 文件（每个“页面”一个 HTML）                    | 只有一个主要的 HTML 文件                                       |
| **HTML 来源**      | 每次跳转都需要请求一个新的 HTML 页面                          | 初始加载一个骨架 HTML，后续内容由 JS 动态生成和注入            |
| **路由控制**       | 由后端负责，URL 对应服务器上的具体物理文件资源或接口                       | 由前端 JavaScript 负责（客户端路由）                           |
| **页面跳转刷新**     | 跳转需要整页请求展示，有明显白屏/闪烁                                | 无整页刷新，内容在当前页内动态更新                             |
| **初始加载时间**   | 通常较快（HTML 直接渲染），但取决于页面复杂度                          | 可能较慢（需加载核心 JS/CSS 后才能渲染内容）                   |
| **后续交互速度**   | 较慢（每次交互可能需要请求新页面）                                     | 较快（仅需请求数据，本地更新 DOM）                             |
