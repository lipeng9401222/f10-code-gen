# 文档更新提示词

> 用于检查并更新 vue-docs 文档库索引和经验文档

---

## 提示词模板

```
请检查并更新 vue-docs 文档库

## 检查任务

### 1. 确认更新方式

请用户确认使用哪种方式检查变更：

**方式一：基于 Git 变更状态（推荐）**
- 优点：精确定位新增/删除/修改的文件，减少无效检查
- 适用：文档构建后，vue-docs 目录有 git 变更

**方式二：全量检查**
- 优点：不依赖 git，适合 git 状态不准确的情况
- 适用：手动修改文档、git 变更已提交等情况

**请用户确认**：
- 如果 vue-docs 目录有未提交的变更，推荐使用方式一
- 回复"使用 git"或"全量检查"选择方式

---

### 2. 获取变更信息

#### 方式一：Git 变更

执行 git 命令获取变更文件：

```bash
git status --porcelain vue-docs
```

状态说明：
- `A` = 新增文件
- `D` = 删除文件
- `M` = 修改文件
- `R` = 重命名文件

#### 方式二：全量扫描

扫描各目录统计文档数量：

```bash
# PowerShell 命令统计文件数量
(ls vue-docs/components/components-*.md).Count
(ls vue-docs/ep-components/components-*.md).Count
(ls vue-docs/api/api-*.md).Count
(ls vue-docs/getting-started/getting-started-*.md).Count
(ls vue-docs/guides/guides-*.md).Count
(ls vue-docs/guide/guide-*.md).Count
(ls vue-docs/page-examples/**/*.vue).Count
```

对比索引文件中的记录，找出差异。

---

### 3. 分析变更影响

根据变更情况，分析需要更新哪些索引：

| 变化类型 | 影响范围 | 需要更新的索引 |
|----------|----------|----------------|
| 新增/删除 components/*.md | 组件数量变化 | index.md（根目录）、INDEX.md、components/index.md |
| 新增/删除 ep-components/*.md | 企业组件变化 | index.md（根目录）、INDEX.md |
| 新增/删除 api/*.md | API 文档变化 | index.md（根目录）、INDEX.md |
| 新增/删除 getting-started/*.md | 快速上手变化 | index.md（根目录）、INDEX.md |
| 新增/删除 guides/*.md | 进阶指南变化 | index.md（根目录）、INDEX.md |
| 新增/删除 guide/*.md | 基础指南变化 | index.md（根目录）、INDEX.md |
| 新增/删除 page-examples/**/*.vue | 页面模板变化 | page-examples/page-template-index.md |
| 新增 common/*.md | 公共规范变化 | INDEX.md |
| 修改根目录经验文档 | 内容更新 | 检查是否需要同步更新相关索引 |

---

### 4. 更新 index.md（知识库总索引）

**⚠️ 位置：项目根目录 `vue-docs-for-ai/index.md`**

如有组件/API文档变更，检查并更新以下内容：

| 检查项 | 定位方式 | 更新内容 |
|--------|----------|----------|
| 知识库概览 | 找到 `## 📋 知识库概览` | 更新组件数量统计（如"70+ UI 组件"） |
| 基础组件库分类 | 找到对应的组件分类段落 | 根据新增/删除组件添加/删除链接行 |
| 表单组件列表 | 找到 `### 表单组件` | 添加/删除表单组件链接 |
| 数据展示列表 | 找到 `### 数据展示` | 添加/删除数据展示组件链接 |
| 导航组件列表 | 找到 `### 导航组件` | 添加/删除导航组件链接 |
| 反馈组件列表 | 找到 `### 反馈组件` | 添加/删除反馈组件链接 |
| 其他组件列表 | 找到 `### 其他组件` | 添加/删除其他组件链接 |
| 上传组件列表 | 找到 `### 上传组件` | 添加/删除上传组件链接 |
| 框架级组件 | 找到 `## 🔧 框架级组件` | 添加/删除企业级组件链接 |
| API 文档 | 找到 `## 📚 API 文档` | 添加/删除 API 文档链接 |
| 开发指南 | 找到 `## 📖 开发指南` | 添加/删除指南文档链接 |
| 入门教程 | 找到 `## 📖 入门教程` | 添加/删除入门文档链接 |

**📌 新增组件链接格式**：
```markdown
- **[components-xxx.md](./vue-docs/components/components-xxx.md)** - Xxx 组件说明
```

**📌 新增 API 链接格式**：
```markdown
- **[api-xxx.md](./vue-docs/api/api-xxx.md)** - Xxx API 说明
```

### 9. 报告检查结果（必须确认后再更新）

**⚠️ 重要：检查完成后必须报告结果等待用户确认，不能直接执行更新操作！**

输出格式：
1. 列出变更信息（git 变更文件列表 或 全量扫描差异）
2. 分析每项变更对索引的影响
3. 列出需要更新的索引文件及具体更新内容
4. 列出不需要更新的文件及原因
5. **等待用户确认**："以上内容是否需要更新？请确认后再执行"
6. 用户确认后才能执行更新操作
